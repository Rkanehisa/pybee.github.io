Beyond Web 2.0 - Django and Python in the modern web ecosystem

by Russell Keith-Magee; published 1 April 2016

Django tem mais de 10 anos de idade; mas a web em que ele foi construido - o mundo da "Web 2.0" - não existe mais. Sim, nós ainda precisamos desenvolver rapidamente database-backed websites, AJAX-enabled, porém a web moderna enfrenta novos problemas e desafios. Muitos desses desafios envolvem interação com dispositivos que não são máquinas desktop e plataformas que não utilizam um browser tradicional - lugarees onde o Python não é um didadão de primeira-classe.

Isto significa que temos que abandonar o Django e o Python para usar outras linguagens e frameworks? Nós podemos continuar utilizando Python e Django do lado do servidor para interagir com frameworks independentes do lado do cliente? Ou nós podemos utilizar Python and Django nesses novos ambientes?

Nessa conversa, Dr Russell Keith-Magee explora os problemas e as possíveis soluções. In this talk, Dr Russell Keith-Magee will explore the problems, and the potential solutions, to these problems.

Python on the Move: The state of mobile Python

by Russell Keith-Magee; published 2 August 2015

Nos ultimos 5-10 anos a separação "desktop/servidor" que dominou a computação nos anos 80, 90 e no início dos anos 2000 foi jogado para o terceiro plano - devido os dispositivos mobiles como smartphones e tablets.

Então, qual é a história do Python nesses dispositivos? Embora não se tenha muito a dizer do passado, a boa notícia é que no ultimo ano houve mudanças significativas no horizonte e agora Python no Mobile está a beira de se tornar uma realidade.

Nessa palestra, Russell Keith-Magee vai responder o estado atual de usar Python nos dispositivos móveis, o trabalho que está sendo feito, as áreas onde mais trabalho é necessário e as perspectivas do futuro.

Tinkering with Tkinter

by Russell Keith-Magee; published 10 July 2013

Tkinter - o wrapper Python para a biblioteca gráfica do TK - é parte da biblioteca padão desde as primeiras versões. No entanto, essa inclusão não se traduziu em uso extensivo.

Existem boas razões para isso. A documentação do TK é horrível. E caso você consigua superar essa dificuldade, os aplicativos do Tkinter parecem horriveis - ele é repleto de widgets inadequados, com o melhor que o estilo gráfico do open source dos anos 90 tem para oferecer.

E então, o mundo se tornou obsecado com os frameworks web, e o desktop foi declarado morto.

Entretanto, nos ultimos anos, muitas das razões do Tkinter ser ignorado foi lentamente corrigida. Tk 8.4 melhorou imensamente a aparência visual do Tk, surgiu o tkdocs.com, corrigindo muitos dos problemas com a documentação do Tk.

Nessa palestra, você vai ser reintroduzido a um velho amigo e uma explicação do porque, em um mundo web e mobile, você deve se importar com isso.
