+The best place to start with any open source contribution is with something that fits your knowledge, experience, and area of interest.
+
+Python
+-------
+
+If you're a Python programmer, and like the idea of a very well specified, but complex problem, `Colosseum <https://github.com/pybee/colosseum>`__ is a good place to contribute. Colosseum is a pure-Python, browser implementation of the CSS layout algorithm. BeeWare uses it lay out widgets on the screen of an application. The CSS specification has an extensive test suite; Colosseum needs to pass all those tests.
+
+The Colosseum documentation has a `contribution guide <http://colosseum.readthedocs.io/en/latest/how-to/contribute.html>`__ that walk you through your first Colosseum contribution.
+
+GUI programming
+----------------
+
+If you've got experience with a native widget library - Cocoa on macOS, GTK+ on Linux, Windows Forms, or the native iOS or Android libraries, you may be able to help with `Toga </project/projects/libraries/toga/>`__. Toga is a cross platform widget toolkit - it provides a common interface for user interface elements like buttons and trees.
+
+Toga has a range of interfaces already defined; however, the implementation of those interfaces isn't complete across all platforms. Pick a widget, and implement that widget for your platform of choice!
+
+Even if you *don't* know a native widget library, you may still be able to help with Toga. Toga's core library has a test suite, but the coverage of that test suite isn't very good. Pick a widget interface, and see if you can write a test for it!

 Windows
 ---------

Android e Java

O melhor lugar para começar com qualquer contribuição open source é com algo que se encaixe com sua área de interesse. Se você tem um telefone Android, ou você está interessado em desenvolver código que precisam rodar na JVM, VOC é um fantástico lugar para contribuir. Para VOC poder funcionar, toda a funcionalidade do CPython que está implementada em C precisa ser reimplementada em Java. Desta forma, existem muitas tarefas acessíveis e bem delimitadas que são apropriadas para novos contribuidores.

Veja o guia para sua primeira contribuição VOC para mais detalhes.

JavaScript e Navegadores

Outra possibilidade, se você é um desenvolvedor web que gostaria de usar Python no navegador, Batavia foi feito para você. Assim como VOC, Batavia requer uma implementação de todas as funcionalidades do CPython que estão implementadas em C; no entanto, Batavia precisa dessa implementação em JavaScript.

Novamente, isso resulta em muitas tarefas acessíveis e bem delimitadas que são apropriadas para novos contribuidores.

Veja o guia sua primeira contribuição ao Batavia para mais detalhes.

Tutoriais

Uma das melhores formas de contribuir é garantir que nossa documentação de introdução e o tutorial é consistente. Escolha um projeto do seu interesse da BeeWare suite, e tente seguir o guia de como começar e o tutorial. Se você não conseguir encontrar o guia de como começar - então essa é a primeira coisa a reportar!

Conforme você utilizar o tutorial, tome nota de qualquer coisa que não faça sentido, ou não funciona. Então abra um bug report para esses problemas.

Windows

Se você é um usuário Windows, BeeWare pode definitivamente utilizar sua ajuda. Todo o time desenvolvedor utiliza Linux e MacOS no seu trabalho diário, e como resultado, é realmente fácil termos incompatibilidades com Windows. Isso é quase sempre graças a um engano. Identificando - e até mesmo consertando - problemas com as ferramentas de BeeWare no Windows é de imensa ajuda.

Línguas

Você fala uma outra língua além do Inglês? Atualmente, o time desenvolvedor é composto inteiramente por nativos em Inglês. Prover traduções da documentação, ou tutoriais inteiramente completos ou guias na sua língua nativa é uma grande forma de contribuir. Escolha um projeto que já tenha documentação e traduza o texto. Outra possibilidade, é começar a escrever do zero um tutorial inteiramente novo.

Se você fala uma língua com um alfabeto não-latino (ex: Russo ou Chinês), ou uma língua com um sistema de escrita da direita para a esquerda (ex, Hebraico ou Árabe) você pode apontar todos os lugares onde nós decidimos utilizar codificação ASCII ou escrita esquerda-direita. Escolha qualquer dos projetos ativos BeeWare e tente usá-lo com sua língua nativa. Se você tiver qualquer problema, reporte esses problemas como bugs; se você estiver se sentindo particularmente motivado, veja se você pode também ajudar a consertar o bug.
